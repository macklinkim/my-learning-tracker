# 개발 진행 과정 (Development Phases)

## Phase 6: 확장 기능 및 LMS 특화 UX 구현 (Extra Features)

**목표:** 유명 LMS 및 생산성 앱들의 핵심 기능을 마이크로 러닝 트래커에 이식하여, 동기 부여 및 사용성을 극대화합니다.

### 상세 작업 내용
- **학습 연속성 시각화 (Activity Heatmap 및 Streak)**
  - **출처**: GitHub, LeetCode, 백준(Baekjoon)
  - **구현 방식**: `react-calendar-heatmap` 또는 `shadcn/ui` 기반의 커스텀 컴포넌트를 사용하여 GitHub이나 LeetCode와 같이 1년 365일을 작은 사각형(그리드)으로 시각화합니다.
  - **상세 기능**: 
    - 하루 동안의 학습 완료량(또는 집중 시간)에 따라 색상의 농도를 다르게 표현합니다 (예: 연한 초록색부터 진한 초록색까지). 달성값이 0이면 회색으로 표시합니다.
    - 각 그리드(사각형)에 마우스를 올리면(Hover Tooltip), 해당 날짜의 총 학습 건수와 주요 학습 카테고리를 직관적으로 보여줍니다.
    - 백엔드에서 'Streak(연속 학습일)' 계산 로직을 구현하여 "현재 5일 연속 학습 중!" 같은 뱃지를 부여해 강력한 동기 부여를 제공합니다.
  
- **인박스 스크랩북 (Inbox Scrapbook) 구현**
  - **출처**: Notion Web Clipper, Pocket
  - 분류되지 않은 학습 자료(URL)를 일단 임시 저장소(인박스)에 던져놓을 수 있는 독립 페이지 및 UI를 엽니다.
  - 클립보드에 있는 URL 주소를 쉽게 붙여넣기 및 제목 자동 파싱, 목록으로 바로 노출.

- **학습 로드맵 시각화 및 드래그 앤 드롭 분류 확장 (Learning Path)**
  - **출처**: Udemy, 인프런
  - 인박스에 쌓인 아이템을 학습 로드맵 카테고리에 편하게 할당할 수 있도록, `dnd-kit`을 활용한 복합 드래그 앤 드롭 기능을 만듭니다.
  - 관련 학습 아이템들을 선행/후행 순서 의존성에 맞춰 묶어서 "로드맵 뷰(Roadmap Tree)" 컴포넌트로 보여줌으로써 효과적인 방향성을 제시합니다.

- **자동 메타데이터 스크래핑 (Auto Metadata Extraction)**
  - **출처**: Notion, Readwise
  - **기능**: 사용자가 테이블이나 입력창에 URL(유튜브, 블로그, 공식 문서 등)만 붙여넣으면, Hono 백엔드가 즉시 해당 URL을 파싱하여 제목, 썸네일 이미지(Open Graph 태그), 예상 읽기 시간 등을 자동으로 채워주는 기능입니다.
  - **장점**: 백엔드의 비동기 처리 및 크롤링 역량을 보여주기에 완벽하며, 사용자는 일일이 제목을 직접 칠 필요가 없어 사용자 경험(UX)이 극대화됩니다.
